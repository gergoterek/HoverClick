## 1. Problem Statement

HoverClick focuses and raises background windows before delivering the original click.
This is the desired behavior for most windows, but it breaks certain apps where the
click must reach the original target without any focus-steal occurring first.

The concrete known case is **Maccy** (clipboard manager): when the user clicks a
Maccy history item, Maccy pastes the selected text into whatever app was frontmost
before Maccy opened. If HoverClick steals focus before the click is delivered, paste
lands in the wrong target.

v1.2.0 shipped a built-in Maccy bypass. The `feature-compatibility-bypass-followup`
branch (merged at `5dd9446`) refactored the bypass check into a centralized
`isCompatibilityBypassApplication:` method, making it easier to extend.

**The user has now clarified that v1.3 must include a proper Excluded Apps feature,
not only a hard-coded Maccy bypass.** Users need to be able to exclude additional
apps from HoverClick's focus behavior.

---

## 2. User Requirement

> "v1.3 must include Excluded Apps, not only Maccy compatibility."

The Excluded Apps feature must allow users ‚Äî not only developers ‚Äî to control which
apps HoverClick skips. Maccy is the first and currently only built-in bypass, but
other clipboard managers, launcher apps, screenshot tools, and similar utilities may
need the same treatment.

---

## 3. Current v1.2.0 / post-5dd9446 Maccy Behavior

- `isCompatibilityBypassApplication:` checks bundle ID `org.p0deje.Maccy` first,
  then falls back to app name `"Maccy"` if the bundle ID is empty.
- When a click's target app matches, HoverClick skips focus/raise/activate entirely.
- The original event is returned unchanged.
- `_lastBypassDecision` is set to `"bypassed-maccy"`.
- Copy Diagnostics Summary shows `Last bypass decision: bypassed-maccy`.
- This list is **hard-coded in source**; users cannot add or remove apps.

---

## 4. Why the Narrow Maccy-Only Bypass Is Insufficient

- Only one app is handled.
- Users with other clipboard managers (Pasta, ClipMenu, Clipboard Manager, etc.)
  or launcher apps (Alfred, Raycast, etc.) face the same problem with no recourse.
- The bypass is invisible to users ‚Äî there is no UI indicating which apps are excluded.
- Users cannot inspect the list, add new apps, or remove Maccy if they want it
  focused normally.
- The feature-compatibility-bypass-followup refactor made the code cleaner but did
  not change the fundamental limitation: the list is not user-configurable.

---

## 5. Product Definition

An **Excluded App** in HoverClick is any app whose click events HoverClick must
pass through **without attempting focus/raise/activate on any target window**.
When a click's target resolves to an excluded app:

- HoverClick performs no AX hit-testing, no focus, no raise, no activation.
- The original event is returned unchanged.
- A diagnostic record is written for the bypass.

Excluded Apps are distinct from the modifier Bypass Key:

| Mechanism | Who configures it | When it fires |
|---|---|---|
| Bypass Key (Shift / Fn) | User per click, via held modifier | Before target is resolved |
| Excluded Apps | User per app, via settings | After target is resolved |

Maccy is the only **built-in** excluded app (hardcoded, always active).
The user list is a **configurable layer on top**.

## 6. Non-Goals

The following are explicitly out of scope for the v1.3 Excluded Apps MVP:

- Broad app picker UI with NSOpenPanel or Applications folder browser.
- Per-app focus rules (partially exclude, redirect, delay).
- Whitelisting specific windows within an excluded app.
- Automatic detection / suggestion of apps to exclude.
- Cloud sync or iCloud storage of the exclusion list.
- CDMBA / Click-Time Override.
- Hover Click Assist.
- Scroll Focus.
- Finder selection hack.
- Background drag / text-drag fix.
- Tooltip work.
- Any event tap mask change.
- Any synthetic clicks, event replay, delayed click delivery, cursor movement,
  or CGEventPost.
- Any signing, bundle ID, version/build, appcast, or release change in this plan.

| Dimension | Assessment |

### Option D ‚Äî Hybrid MVP: built-in Maccy + simple submenu + deferred app picker

Implement Option B now (user-configurable list via bundle ID text entry, simple
submenu UI). Document the app picker as a concrete future enhancement (not deferred

add the picker in a later cycle after validating the storage and bypass architecture.

| Dimension | Assessment |
|---|---|
| User value | High ‚Äî user can exclude any app; Maccy works automatically. |
| Implementation risk | Low-medium ‚Äî builds directly on existing refactored bypass method. |
| Runtime risk | Low ‚Äî bypass check consults NSUserDefaults array; no new event paths. |
| UI complexity | Low ‚Äî submenu + NSAlert; no custom windows or panels. |
| Persistence complexity | Low ‚Äî NSUserDefaults array of bundle IDs. |
| Manual testing needed | Yes ‚Äî same as Option B, plus regression for Maccy built-in bypass. |
| **Recommendation** | **Recommended MVP.** Balances user value, implementation risk, and forward compatibility. |

## 8. Recommended MVP: Option D (Hybrid)

### Summary

- **Built-in bypass:** Maccy (`org.p0deje.Maccy`) remains hard-coded and always active.
  It is shown as a non-removable built-in entry in the submenu.
- **User list:** Stored as `NSArray<NSString*>` of bundle IDs in `NSUserDefaults`
  under key `excludedAppBundleIDs`.
- **UI surface:** A new `Excluded Apps` submenu inside the Functions section of the
  status menu, between `Bypass Key` and the section separator.
- **Add:** A persistent `Add App...` item opens a small `NSAlert` with a text field.
  The user enters a bundle ID (e.g. `com.runningwithcrayons.Alfred`). HoverClick
  validates the string is non-empty and not already in the list.
- **Remove:** Each user-added entry in the submenu has a secondary row or action
  to remove it. The simplest approach: clicking a user-added entry toggles/removes it.
- **App picker:** Explicitly deferred to a later cycle.

## 9. Data Model Proposal

```
NSUserDefaults key: "excludedAppBundleIDs"
Type: NSArray<NSString*>
Default value: empty array (Maccy is built-in, not stored in user list)
```

**Runtime lookup:**

```
- (BOOL)isExcludedApplication:(NSRunningApplication *)app {
    // Built-in: always bypass Maccy
    // (org.p0deje.Maccy or name fallback "Maccy")
    if ([self isBuiltInCompatibilityBypassApplication:app]) return YES;

    // User-configured list: check by bundle ID
    NSString *bundleID = app.bundleIdentifier;
    if (bundleID.length > 0) {
        NSArray<NSString*> *userList = [[NSUserDefaults standardUserDefaults]
                                        arrayForKey:@"excludedAppBundleIDs"] ?: @[];
        return [userList containsObject:bundleID];
    }
    return NO;
}
```

**Notes:**
- The rename from `isCompatibilityBypassApplication:` to `isExcludedApplication:`
  may be appropriate at implementation time, keeping the built-in Maccy check as a
  private helper (`isBuiltInCompatibilityBypassApplication:`).
- Alternatively, retain `isCompatibilityBypassApplication:` as the built-in check
  and add a new `isUserExcludedApplication:` for the user list, calling both from
  the event path. Either approach is acceptable; choose at implementation time based
  on what reads most clearly.
- Do not store Maccy in the user list. The built-in bypass is always active
  regardless of the user list state.

## 11. Runtime Behavior Proposal

The event path in `handleLeftMouseDown:` and `handleRightMouseDown:` already checks
`isCompatibilityBypassApplication:` after target resolution. The extension is:

```
1. [resolve AX target app]
2. if [self isExcludedApplication:targetApp]:
   - set _lastBypassDecision = "excluded-app:<bundleID>"  (or keep "bypassed-maccy" for Maccy)
   - log: "excluded app <bundleID>; skipping focus/raise/activate; original event returned unchanged"
   - record focus decision: skipped, excluded app
   - record click-through investigation
   - include in recent decision history
   - complete recent decision: "skipped: excluded app (<bundleID>)"
   - set last click result: "Excluded App (<bundleID>)"
   - return (original event returned unchanged by callback)
3. [continue normal focus path]
```

**Diagnostic string for built-in Maccy bypass:** Retain `"bypassed-maccy"` to
preserve test matrix expectations, unless a deliberate decision is made to migrate
to `"excluded-app:org.p0deje.Maccy"` and update the test matrix at the same time.
The implementation prompt should make this decision explicit.

**Diagnostic string for user-added apps:** Use `"excluded-app:<bundleID>"` to
distinguish user-configured bypasses from the built-in Maccy bypass.

**Event semantics remain unchanged:**
- No synthetic clicks.
- No event replay.
- No delayed delivery.
- No cursor movement.
- No CGEventPost.
- Original event returned unchanged in all bypass paths.
- Event tap mask: `kCGEventLeftMouseDown | kCGEventRightMouseDown` only.

## 14. Safety Constraints

These apply during the implementation phase and must be verified before merge:

- Event tap mask must remain exactly `kCGEventLeftMouseDown | kCGEventRightMouseDown`.
- All bypass paths must return the original event unchanged.
- `NULL` must not be returned for normal user events.
- No synthetic click, event replay, delayed click delivery, cursor movement, or
  CGEventPost in any new code path.
- No new event types tapped.
- No mouse-move, mouse-dragged, mouse-up, scroll, keyboard, or flagsChanged tap.
- No AutoRaise or hover-to-focus behavior.
- No change to signing identity, bundle ID, Info.plist, or version/build metadata.
- `scripts/ci-safety-check.sh` must pass.
- `scripts/build-app.sh` must succeed with no compiler warnings beyond existing ones.
- `scripts/verify-app.sh` must pass (signing, bundle ID, Sparkle, process count).
- The excluded-app check must fire after AX target resolution, not before it (to keep
  the bypass scoped to actual target-app identity, not click location).
- The built-in Maccy bypass must remain active even if `excludedAppBundleIDs` is empty.

## 22. Implementation Addendum (feature-excluded-apps follow-up)

After the initial MVP shipped on `feature-excluded-apps` and passed first manual review, the user requested a native app selector so bundle IDs do not have to be typed by hand, and reported that Cmd+V paste did not work in the manual bundle-ID input. A first follow-up added an `NSOpenPanel` chooser (`Choose Application...`), kept a renamed manual `Add by Bundle ID...` fallback with an `HoverClickEditableTextField` Cmd+V fix.

## 23. UX Revision ‚Äî LinearMouse-style App List Selector

The user reviewed the `NSOpenPanel` chooser and rejected it as the final UX: they do not want a Finder/file-browser flow, do not want to deal with bundle IDs, and do not want a hardcoded visible `Maccy (built-in)` row (which is odd especially when Maccy is not installed). This revision (same branch, same feature) addresses that:

- **App list selector replaces the OpenPanel.** `Configure for...` now opens a dedicated list selector ‚Äî an `NSAlert` with an `NSPopUpButton` of installed apps (display name + icon), sorted alphabetically and deduplicated by bundle ID (LinearMouse "Configure for" spirit). It is a list picker, not a file browser.
- **App enumeration is bounded.** `installedApplicationsForSelector` scans only `/Applications`, `/Applications/Utilities`, `/System/Applications`, `/System/Applications/Utilities`, and `~/Applications`, shallow (no full-disk scan, no deep recursion). Display name from `CFBundleDisplayName`/`CFBundleName`/file name; bundle ID from `NSBundle bundleWithURL:`; icon from `NSWorkspace iconForFile:`.
- **No more bundle IDs in the normal UI.** `Add by Bundle ID...`, the `HoverClickEditableTextField` subclass, the `Choose Application...` OpenPanel, and the `panel:shouldEnableURL:` delegate were removed. The full Applications-browser/edge-case picker from Option C remains deferred.
- **Maccy row removed from the menu.** The built-in Maccy compatibility bypass stays internal and always active, but is no longer shown as a fixed visible row. If the user picks Maccy from the selector, it is not duplicated into the user list; a friendly "already handled automatically" message is shown.
- **Friendly names in the submenu.** Stored bundle IDs are resolved to display names via `displayNameForExcludedBundleID:` (`NSWorkspace URLForApplicationWithBundleIdentifier:`), falling back to the bundle ID when the app is not installed.
- **Diagnostics wording softened.** The built-in line now reads `Excluded Apps (built-in compatibility): Maccy (org.p0deje.Maccy) installed|not installed`.
- **Persistence unchanged.** Key `excludedAppBundleIDs`, `NSArray<NSString *>`, no migration or reset. The selector saves via the shared `addExcludedBundleID:` (rejects missing IDs, duplicates, and built-in Maccy).
- **Safety unchanged.** Event tap mask, original-event pass-through, and the no-synthetic-click/no-replay/no-cursor-movement constraints are unchanged.
