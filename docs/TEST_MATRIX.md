## Stable Checkpoint Manual Validation

Manual Finder UI validation -- not run automatically.

The current stable checkpoint has been manually validated with:

- Left Click Focus ON focuses background windows and returns the original left-click event unchanged.
- Left Click Focus OFF passes left-clicks through without running the focus path.
- Right Click Focus ON focuses the target background window first and returns the original right-click event unchanged so the normal context menu works.
- Right Click Focus OFF passes right-clicks through without running the focus path.
- Finder context-menu follow-up left-click passes through without AX hit-testing after a recent Finder right-click while Finder is frontmost.
- Bartender/menu-bar overlay clicks pass through without focusing the window behind the overlay.
- The HoverClick status menu, Diagnostics submenu, Copy Diagnostics Summary, Permissions & Startup submenu, Open Accessibility Settings action, and Quit/Cmd+Q remain present and uncluttered.
- Launch at Login works.
- Mouse movement alone does not focus windows.
- Known limitations are documented as limitations, not failures: Finder right-click actual selection may not change, and background text first-drag can require a second drag in some apps.
- Accessibility permission did not reappear.
- No synthetic click, cursor movement, `CGEventPost`, `CGDisplayMoveCursorToPoint`, `kCGEventMouseMoved`, or scroll-focus behavior exists.
